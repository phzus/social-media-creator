# Delivery Standards — Marketing Squad Extremo

## REGRA INEGOCIAVEL: Toda Entrega em HTML Visual

**Severidade: NON-NEGOTIABLE**

Toda entrega de qualquer agente que produza conteudo em .md (copy, briefings, estrategias, relatorios, scripts, sequencias de email, etc.) DEVE ser convertida em HTML visual antes de ser entregue ao Anthony.

## Por Que HTML?

Anthony faz demonstracoes ao vivo para potenciais clientes exigentes. Os documentos precisam:
- Ser visualmente impactantes e profissionais
- Parecer entregas de agencia premium (8 digitos de faturamento)
- Surpreender e impressionar clientes criticos
- Nunca parecer output de IA ou documento cru

## Pipeline de Entrega Obrigatorio

```
AGENTE PRODUTOR (.md) --> @theboss (revisao anti-IA) --> @picasso (direcao criativa HTML) + @pixel (implementacao HTML) --> ENTREGA FINAL (.html)
```

### Fluxo Detalhado

1. **Agente produtor** (ex: @claudinho) cria o conteudo em .md
2. **@theboss** revisa o conteudo (anti-IA + alinhamento estrategico)
3. **@picasso** define a direcao criativa do HTML (layout, cores, tipografia, imagens)
4. **@pixel** implementa o HTML visual seguindo a direcao do @picasso
5. **@theboss** faz revisao final do HTML (qualidade visual + anti-IA)
6. **Entrega** ao Anthony em .html pronto para apresentacao

### Requisitos do HTML Final

- **Layout profissional** com hierarquia visual clara
- **Tipografia premium** (fontes distintas, nunca genericas)
- **Imagens reais** (Unsplash/Pexels) — NUNCA emojis, icones ou vetores basicos
- **Banners e headers** visuais impactantes
- **Efeitos CSS** sofisticados (gradientes, sombras, glassmorphism)
- **Responsivo** (desktop + mobile)
- **Cores do brandbook** do projeto (quando existir)
- **Menos de 60% texto** — o resto com imagens, backgrounds, efeitos
- **Zero cara de IA** — deve passar no Teste da Agencia

### Excecoes (NAO precisam de HTML)

- Arquivos internos de configuracao (.yaml, .json)
- Handoff artifacts entre agentes
- Delegacoes internas do @theboss
- Checklists operacionais internos

## Nomenclatura

```
{PROJETO}_{FASE}_{TIPO}_{CANAL}_{DESCRICAO}_V{N}.html
```

Mesma taxonomia dos .md, mas com extensao .html.

## Regra de Versionamento

- Toda versao .html acompanha a versao .md correspondente
- Ambos os arquivos sao salvos (md como fonte, html como entrega)
