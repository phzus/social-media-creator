# Quality Gates — Marketing Squad Extremo

## Gates de Qualidade

Quality gates são pontos de verificação obrigatórios que BLOQUEIAM o avanço se não forem cumpridos.

## Gates por Fase do Ciclo de Vida

### Gate 0: Pesquisa de Mercado Completa
- **Quando:** ANTES de qualquer outra fase — e o primeiro passo de todo projeto
- **Quem verifica:** @theboss + Anthony
- **Criterios:**
  - [ ] @analyzer executou pesquisa em 8 frentes paralelas
  - [ ] Relatorio de Mercado completo com dados reais e fontes
  - [ ] Analise de concorrencia com todos os concorrentes listados
  - [ ] Avatar/persona mapeado com dores, desejos e linguagem reais
  - [ ] Conteudos hypados e tendencias identificados
  - [ ] Objecoes mapeadas com estrategias de quebra
  - [ ] Benchmarks de metricas do nicho documentados
  - [ ] Gaps e oportunidades identificados
  - [ ] Relatorio entregue em HTML visual
  - [ ] @theboss revisou + Anthony aprovou
- **Se falhar:** @analyzer complementa pesquisa (max 3 ciclos)

### Gate 0.5: Instilacao de Tom de Voz
- **Quando:** Apos pesquisa de mercado, ANTES do briefing
- **Quem verifica:** @theboss + Anthony
- **Criterios:**
  - [ ] Materiais do expert coletados (minimo 5 pecas de conteudo)
  - [ ] Analise profunda de vocabulario, girias, jargoes, tom, ritmo
  - [ ] COMUNICACAO.md completo com todas as 10 secoes
  - [ ] Exemplos reais do expert incluidos (trechos anotados)
  - [ ] Anti-patterns documentados (o que o expert NUNCA diria)
  - [ ] Guia de aplicacao por formato (email, pagina, WhatsApp, ad, VSL)
  - [ ] Entregue em HTML visual
  - [ ] @theboss revisou + Anthony aprovou
- **Se falhar:** @analyzer complementa analise com mais materiais

### Gate 2.5: Brandbook Aprovado
- **Quando:** Antes de montar squad / iniciar qualquer producao visual
- **Quem verifica:** Anthony

  - [ ] Minimo 4 propostas de logo apresentadas ao cliente

### Gate 4: Copy Revisada
- **Quando:** Antes de publicar qualquer peça de copy
- **Quem verifica:** @theboss (usando copy-review checklist)
- **Critérios:**
  - [ ] Score >= 75% no checklist de revisão
  - [ ] Tom de voz consistente com COMUNICACAO.md
  - [ ] Nenhuma promessa falsa ou exagerada
  - [ ] CTA claro e específico
  - [ ] Ortografia e gramática OK
  - [ ] **TESTE ANTI-IA APROVADO** (3 perguntas bloqueantes — ver anti-ai-protocol.md)
- **Se falhar:** @claudinho corrige com feedback específico (max 3 ciclos)

### Gate 4.5: Anti-IA Compliance
- **Quando:** Antes de QUALQUER entrega (copy, criativos, paginas, relatorios)
- **Quem verifica:** @theboss
- **Critérios:**
  - [ ] "Isso parece que foi a IA que fez?" — NAO
  - [ ] "Uma pessoa comum desconfiaria de IA?" — NAO
  - [ ] "Poderia ter saido de um template do ChatGPT?" — NAO
  - [ ] Zero palavras/expressoes da blacklist anti-IA
  - [ ] Tom natural, humano, com personalidade do expert
  - [ ] Visual sem emojis, icones genericos, layouts commodity
- **Se falhar:** Volta para agente produtor com feedback especifico. BLOQUEANTE.

### Gate 4.7: HTML Visual Delivery
- **Quando:** Antes de entregar qualquer documento ao Anthony
- **Quem verifica:** @theboss
- **Critérios:**
  - [ ] Entrega convertida de .md para .html visual pelo @picasso + @pixel
  - [ ] HTML com layout profissional, tipografia premium, imagens reais
  - [ ] Menos de 60% texto — resto com imagens, efeitos, backgrounds
  - [ ] Zero emojis, icones ou vetores basicos
  - [ ] Banners e headers visuais impactantes
  - [ ] Nivel de qualidade: agencia premium de 8 digitos
  - [ ] Pronto para apresentacao ao vivo em pitch de vendas
- **Se falhar:** @pixel ajusta com direcao do @picasso (max 3 ciclos)

### Gate 6: Aprovação Final

  - [ ] Todas as peças revisadas e aprovadas por @theboss

## Gates Específicos por Tipo de Funil

### PLF (Lançamento)
- **Antes do Evento:** CPLs prontos, sequência de emails agendada, slides finalizados
- **Antes do Carrinho:** Página de vendas testada, checkout OK, equipe de vendas briefada

### Webinar
- **Antes do Webinar:** Script ensaiado, slides prontos, plataforma testada, teste de áudio/vídeo

### Desafio
- **Antes do Dia 1:** Conteúdo de TODOS os dias prontos, grupo criado, automações ativas

### High Ticket
- **Antes das Calls:** Script validado com role-play, pipeline com volume mínimo

### Evento Presencial
- **4 sem antes:** Local confirmado + oferta aprovada
- **1 sem antes:** Inscritos >= meta mínima + materiais prontos
- **Dia do evento:** A/V testado + equipe nos postos

## Regras de Enforcement

1. **Gates são BLOQUEANTES** — não há exceção
2. **Ninguém pula gate** — nem Anthony (ele pode APROVAR com ressalvas, mas não pular)
3. **Gate falhou = volta** — sempre volta para a fase anterior
4. **Máximo 3 tentativas por gate** — no 3º, escalar para Anthony decidir
5. **Gates são documentados** — registro de quem aprovou, quando, com que condições
