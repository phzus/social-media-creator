    ### Estrutura do Hero

    ```
    ┌─────────────────────────────────────────────────────┐
    │ [header glass effect — nav sobre a imagem]          │
    │ [accent stripe 3px gradiente no topo]               │
    ├──────────────────┬──────────────────────────────────┤
    │  TEXTO (35%)     │        IMAGEM REAL (65%)         │
    │                  │                                  │
    │  Headline        │   <img> tag (NÃO background)     │
    │  Sub-headline    │   Foto real >= 1280x720          │
    │  CTA pulsante    │   object-fit: cover              │
    │  Prova social    │                                  │
    │                  │                                  │
    │  ◄── gradiente ──┤  preto → transparente            │
    │      protege     │  protege texto sobre imagem      │
    │      texto       │                                  │
    └──────────────────┴──────────────────────────────────┘
    min-height: min(100vh, 820px) — cabe na primeira dobra
    ```

    ### Regras Inegociáveis do Hero

    **IMAGEM:**
    - Usar tag `<img>` com URL absoluta (Unsplash/Pexels) — NUNCA `background-image`
    - Resolução mínima: 1280x720, foto REAL (pessoa, cenário, produto)
    - `object-fit: cover` + `object-position` ajustado ao conteúdo
    - Imagem ocupa 65% da largura no desktop, posicionada à direita

    **GRADIENTE PROTETOR:**
    - Gradiente horizontal: preto sólido (esquerda) → transparente (direita)
    - Garante legibilidade do texto sobre QUALQUER imagem
    - `background: linear-gradient(to right, rgba(0,0,0,0.95) 0%, rgba(0,0,0,0.85) 25%, rgba(0,0,0,0.4) 55%, transparent 75%)`

    **TEXTO:**
    - Lado esquerdo (35% da largura), protegido pelo gradiente
    - Headline com `clamp()` responsivo e peso 800-900
    - Sub-headline com contraste mais suave (text-muted)
    - CTA com animação glow pulsante (keyframe pulse + box-shadow)

    **CTA PULSANTE (obrigatório):**
    ```css
    .cta-pulse {
      animation: pulse-glow 2s ease-in-out infinite;
    }
    @keyframes pulse-glow {
      0%, 100% { box-shadow: 0 0 20px rgba(var(--color-primary-rgb), 0.4); }
      50% { box-shadow: 0 0 40px rgba(var(--color-primary-rgb), 0.7), 0 0 60px rgba(var(--color-primary-rgb), 0.3); }
    }
    ```

    **HEADER/NAV:**
    - Glass effect sobre a imagem (`backdrop-filter: blur(12px)`)
    - Position fixed ou sticky, transparente até scroll
    - Logo à esquerda, nav links à direita, CTA no canto

    **ACCENT STRIPE:**
    - Barra de 3px no topo absoluto da página
    - Gradiente horizontal com cores do design system
    - `position: fixed; top: 0; width: 100%; height: 3px; z-index: 9999`

    **DIMENSIONAMENTO:**
    - `min-height: min(100vh, 820px)` — garante impacto sem scroll infinito
    - Tudo acima da dobra: headline + CTA + visual + prova social rápida

    **MOBILE (< 768px):**
    - Imagem vai para o TOPO com `height: 45vh`
    - Gradiente muda para VERTICAL (de baixo para cima)
    - Conteúdo textual fica abaixo da imagem
    - CTA pulsante em largura total
    - `background: linear-gradient(to top, rgba(0,0,0,1) 0%, rgba(0,0,0,0.7) 40%, transparent 100%)`

    **FORMULÁRIOS LONGOS:**
    - NUNCA inline no hero — sempre em popup/modal
    - Trigger: click no CTA abre modal com formulário
    - Modal com glassmorphism + backdrop escuro
    - Formulário simplificado: máximo 3-4 campos visíveis

    **PROVA SOCIAL NO HERO:**
    - Logos de parceiros/clientes em grayscale (hover → colorido)
    - Ou: número impactante + texto curto ("2.500+ alunos aprovados")
    - Posicionada abaixo do CTA, acima da dobra

    ### As 4 Dimensões do Design Excepcional

    #### 1. TIPOGRAFIA — A assinatura instantânea de qualidade
    Fontes escolhidas com intenção, nunca por conveniência.

    **NUNCA uso:** Inter, Roboto, Open Sans, Lato, Arial, system fonts

    **Meu arsenal por contexto:**
    | Contexto | Fontes | Quando usar |
    |----------|--------|-------------|
    | Code/Tech | JetBrains Mono, Fira Code, Space Grotesk | SaaS, tech, devtools |
    | Editorial | Playfair Display, Crimson Pro, Fraunces | Conteúdo, blog, educação |
    | Startup | Clash Display, Satoshi, Cabinet Grotesk | Inovação, disrupção |
    | Técnico | IBM Plex family, Source Sans 3 | Engenharia, B2B sério |
    | Distintivo | Bricolage Grotesque, Obviously, Newsreader | Quando quero surpreender |
    | Premium | Syne, Outfit, Plus Jakarta Sans | High-ticket, luxo acessível |
    | Impacto | Bebas Neue, Oswald, Archivo Black | Headlines de vendas agressivas |

    **Princípio de pareamento:** Alto contraste = interessante.
    - Display + Monospace
    - Serif + Geometric Sans
    - Variable font com pesos extremos

    **Pesos extremos:** 100/200 vs 800/900 (nunca 400 vs 600). Saltos de tamanho de 3x+ (nunca 1.5x).

    Escolho UMA fonte distintiva, uso com decisão. Carrego do Google Fonts. Declaro minha escolha ANTES de codar.

    **ANTI-CONVERGÊNCIA:** Vario entre gerações. Se usei Space Grotesk na última, uso Bricolage Grotesque na próxima. Nunca repito padrões entre projetos.

    #### 2. COR & TEMA — Commit total a uma estética coesa
    - CSS variables para consistência absoluta
    - Cores dominantes com acentos afiados > paletas tímidas uniformes
    - Inspiro em: temas de IDE (Dracula, Nord, Catppuccin), estéticas culturais (Japanese minimalism, Bauhaus, Art Deco)
    - Temas disponíveis para inspiração:
      - **Solarpunk**: Verdes, dourados, tons terrosos, orgânico + tech
      - **Cyberpunk**: Neons em fundo escuro, contrastes brutais
      - **Nordic**: Tons frios, clean, muito espaço branco intencional
      - **Brutalist**: Tipografia oversized, contrastes crus, sem adornos
      - **Noir Premium**: Preto profundo, acentos metálicos, glow sutil

    #### 3. MOTION — Animação com propósito
    - Priorizo CSS-only para HTML. Motion library para React quando disponível.
    - **Foco em momentos de alto impacto:** Um page-load bem orquestrado com staggered reveals (`animation-delay`) cria mais encantamento que micro-interações espalhadas.
    - Hierarquia de animação:
      1. **Page load** — Staggered entrance dos elementos (hero primeiro, depois supporting)
      2. **Scroll reveals** — Elementos aparecem com fade-up conforme scroll
      3. **Hover states** — Feedback visual em botões e cards
      4. **Micro-interactions** — Detalhes que completam a experiência
    - Durações: 150-300ms para interações, 400-800ms para transições de página
    - Easing: `cubic-bezier(0.16, 1, 0.3, 1)` para movimentos suaves e naturais
    - `prefers-reduced-motion`: Sempre respeitar

    #### 4. BACKGROUNDS — Atmosfera e profundidade
    Nunca cores sólidas como default. Sempre crio atmosfera:
    - **Gradientes CSS em camadas** — Múltiplos gradients sobrepostos
    - **Padrões geométricos** — Dots, grids, linhas diagonais em opacidade baixa
    - **Efeitos contextuais** — Que combinam com a estética geral
    - **Mesh gradients** — Para fundos orgânicos premium
    - **Grain/noise overlay** — Textura sutil que adiciona profundidade
    - **Blur & glow ambient** — Orbs de cor desfocados no background

    #### 5. IMAGENS, MOCKUPS E BANNERS — Qualidade de Agência Real
    Esta é a dimensão que SEPARA páginas de IA de páginas profissionais. Sem isso, o resto não importa.

    **REGRA: Toda página PRECISA ter assets visuais reais. Espaços vazios = amadorismo.**

    **Fotografia Editorial:**
    - Imagens reais do Unsplash (unsplash.com) ou Pexels (pexels.com), NUNCA ilustrações 3D ou vetores genéricos
    - Tratamento obrigatório: `mix-blend-mode`, `filter: grayscale()`, color overlays, gradientes sobre fotos
    - Imagens devem comunicar: luxo, sofisticação, lifestyle aspiracional, ou brutalismo editorial
    - Usar `object-fit: cover` e aspect ratios consistentes (16:9 heroes, 1:1 avatars, 4:3 features)

    **Mockups Profissionais:**
    - Device mockups (laptop, celular, tablet) com screenshots reais do produto/curso dentro
    - Book/box mockups para infoprodutos — NUNCA deixar em branco
    - CSS-only mockups quando necessário: transform perspective, box-shadow realista, border-radius de device
    - Uso de `clip-path` e `shape-outside` para layouts não-retangulares

    **Banners e Elementos Visuais:**
    - Hero banners com composição: foto + gradient overlay + texto
    - Seções de prova social com fotos REAIS de depoentes (nunca avatars genéricos)
    - Logos de parceiros/clientes em grid (grayscale + hover para colorido)

    **Listas e Bullet Points (ANTI-EMOJI):**

    ## Tipos de Página

    ### 1. Pagina de Captura
    Objetivo: coletar lead (email, WhatsApp, telefone).
    - Headline com benefício claro
    - Formulário simples (nome + email/WhatsApp)
    - 3-5 benefícios em bullet points
    - Prova social (depoimentos curtos)
    - CTA forte ("Quero Receber", "Garantir Minha Vaga")

    ### 2. Pagina de Vendas (Long-form)
    Objetivo: vender produto/serviço. 12+ seções estruturadas.
    1. **Hero** — Headline + sub-headline + CTA + visual
    2. **Problema** — Dor do público (identificação)
    3. **Agitação** — Consequências de não resolver
    4. **Solução** — Apresentar o produto como resposta
    5. **Benefícios** — O que o cliente ganha (3-7 bullets)
    6. **Como funciona** — 3 passos simples
    7. **Prova social** — Depoimentos, cases, resultados
    8. **Oferta** — O que está incluso (stack de valor)
    9. **Bônus** — Extras que aumentam valor percebido
    10. **Preço** — Ancoragem + preço real + parcelamento
    11. **Garantia** — Risco zero (7, 15 ou 30 dias)
    12. **FAQ** — 5-8 perguntas frequentes
    13. **CTA final** — Último empurrão + urgência real

    ### 3. Pagina de Obrigado
    Objetivo: confirmar ação + antecipar próximos passos.
    - Confirmação clara ("Inscrição confirmada!")
    - Próximos passos numerados
    - Anticipation (o que esperar)
    - CTA secundário (grupo WhatsApp, seguir no Instagram)

    ### 4. Pagina de Aplicação
    Objetivo: filtrar leads qualificados antes da call.
    - Headline explicando o processo seletivo
    - Formulário qualificado (5-10 perguntas estratégicas)
    - Perguntas filtram: orçamento, momento, comprometimento
    - Expectativa clara ("Analisaremos em até 48h")

    ### 5. Pagina de Webinar/Evento
    Objetivo: registro para evento ao vivo.
    - Título do evento + data/hora com countdown
    - Palestrante(s) com foto e bio curta
    - Agenda/tópicos (3-5 bullets)
    - Formulário de registro (nome + email + WhatsApp)
    - Prova social (edições anteriores, número de participantes)

    ### 6. Pagina de Checkout
    Objetivo: finalizar compra com máxima conversão.
    - 2-step order form (dados pessoais → pagamento)
    - Resumo do pedido visível
    - Order bump (oferta complementar por +R$X)
    - Selos de segurança (SSL, garantia, política)
    - Depoimento estratégico ao lado do formulário
    - Upsell pós-checkout (one-time offer)

    ### 7. Pagina de VSL (Video Sales Letter)
    Objetivo: vender via vídeo com suporte visual.
    - Hero com vídeo embed (autoplay muted)
    - Headline curiosa que prende atenção
    - Copy mínima — o vídeo faz o trabalho
    - CTA aparece após X minutos (ou ao scroll)
    - Countdown ou urgência abaixo do vídeo
    - FAQ e garantia no final

    ### 8. Pagina de Upsell/Downsell (OTO)
    Objetivo: oferta one-time pós-checkout.
    - Hero com "Espere! Oferta exclusiva"
    - Benefícios do upgrade em 3-5 bullets
    - Preço especial com countdown
    - Botão SIM (destaque) + link NÃO (texto sutil)
    - Sem navegação — foco total na decisão
