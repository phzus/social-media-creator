# Task: Criar Engenharia Reversa do Produto

```yaml
task:
  name: criar-engenharia-reversa
  agent: pitch-master
  elicit: true
  output: "engenharia-reversa.md"
```

## Briefing Obrigatório

1. **Produto completo** — Módulos, aulas, conteúdo, bônus
2. **Pesquisa de alunos** — Dores, desejos, objeções (se disponível)
3. **Resultados de alunos** — Top 3 resultados com depoimentos
4. **Promessa principal** — O que o aluno conquista ao final

## Workflow

### Step 1: Mapear o Produto
- Listar todos os módulos/partes
- Identificar a transformação de cada parte
- Marcar os 3 que MAIS geram resultados

### Step 2: Cruzar com Pesquisa
- Top 3 dores da audiência
- Top 3 desejos da audiência
- Objeções mais comuns

### Step 3: Construir Linha do Tempo
```
(SEM RESULTADO) → (CHECKPOINT 1) → (CHECKPOINT 2) → (CHECKPOINT 3) → (PROMESSA)
     CPL 1            CPL 2            CPL 3            PITCH         PRODUTO
```

### Step 4: Definir Narrativa de Cada CPL
- CPL 1: Oportunidade (inconsciente → consciente do problema)
- CPL 2: Como alcançar (consciente da solução)
- CPL 3: Caminho definitivo (consciente do produto)
- Pitch: Oferta (totalmente consciente)

## Output Format

`engenharia-reversa.md` com: mapa do produto, cruzamento com pesquisa, linha do tempo, narrativa por CPL
